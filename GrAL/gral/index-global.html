Introduction to the GrAL

GrAL is the Grid Algorithms Library. It contains generic algorithms and data structures working on grids (or meshes), such as used in finite element modeling, but also in computer graphics, geometric modeling and so on. It does contains some concrete grid data structures, but the focus is clearly on the generic (data-structure neutral) stuff, mainly algorithms.
GrAL's ultimate aim is to provide implementations of algorithms which are as generic as possible , that is, work on any grid data structures satisfying the minimal requirements of the algorithm.

See the file README for some general information. Up-to-date information on GrAL can be found on the GrAL Home page .

Contents of GrAL

GrAL consist of several smaller modules :
  • a growing number (currently 4) of grid-related modules,
  • some low-level support modules, which are not part of GrAL in a proper sense, but are used at some places.
GrAL is very much work in progress. The contents (especially grid-related modules) are not very complete at the moment, and are expected to grow quite a bit.

Using GrAL

First, of course, you have to install GrAL, following the instructions in INSTALL .

If you want to use GrAL, you typically have to adapt your own data structures to satisfy the interface expected by GrAL's generic components. (At present, GrAL does not provide many sophisticated grid data structures.) There is a (still incomplete) tutorial of how to do this. I think that with the help of more elaborate template wrapper, the task of adapting data structures to GrAL can be eased substantially.
