\conceptsection{Cell-On-Cell Iterator Concept}
\end{Label}

\conceptsubsection{Description}
A {\em Cell-On-Cell Iterator\/} refines the concept of 
\sectionlink{\concept{Grid Incidence Iterator }}{GridIncidenceIterator}.
(Strictly speaking, it is not an incidence iterator, but an 
\glossarylink{adjacency}{adjacent}
iterator, because two cells cannot be incident).
It allows to access all cells adjacent to a given cell.

\conceptsubsection{Refinement of}
\sectionlink{\concept{ Incidence Iterator}}{GridIncidenceIterator}
\\
%\sectionlink{\concept{Sequence Iterator}}{GridSequenceIterator}

\conceptsubsection{Notation}
\type{C} is a model of Cell-On-Cell Iterator
\\ 
{\tt c} is an object of type \type{C}
\\
{\tt c1}, {\tt c2} are  objects of type  {\tt C::Cell}

\conceptsubsection{Associated types}
